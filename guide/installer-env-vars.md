# Переменные окружения установщика {#installer-environment-variables}

Установщики Vite+ (`vp-setup.exe`, `install.ps1` и `install.sh`), а также установленная CLI `vp` используют переменные окружения, перечисленные на этой странице.

## Переменные установки {#installation-variables}

Эти переменные управляют работой скриптов установки и автономного установщика для Windows (`vp-setup.exe`).

### `VP_VERSION`

- **Назначение**: версия для установки
- **По умолчанию**: `latest`
- **Эквивалент в CLI**: `--version`
- **Пример**:

  ```bash
  # Unix
  curl -fsSL https://vite.plus | VP_VERSION=1.2.3 bash
  ```

  ```powershell
  # PowerShell
  $env:VP_VERSION = "1.2.3"; irm https://vite.plus/ps1 | iex
  ```

### `VP_HOME`

- **Назначение**: каталог установки; установленная CLI использует эту же переменную в качестве домашнего каталога Vite+ (см. [Переменные окружения](/guide/env))
- **По умолчанию**: `~/.vite-plus` (Unix) или `%USERPROFILE%\.vite-plus` (Windows)
- **Эквивалент в CLI**: `--install-dir`
- **Пример**:

  ```bash
  # Unix
  curl -fsSL https://vite.plus | VP_HOME=/opt/vite-plus bash
  ```

  ```powershell
  # PowerShell
  $env:VP_HOME = "D:\vite-plus"; irm https://vite.plus/ps1 | iex
  ```

### `NPM_CONFIG_REGISTRY`

- **Назначение**: URL пользовательского реестра npm
- **По умолчанию**: `https://registry.npmjs.org`
- **Эквивалент в CLI**: `--registry`
- **Пример**:
  ```bash
  curl -fsSL https://vite.plus | NPM_CONFIG_REGISTRY=https://registry.npmmirror.com bash
  ```

### `VP_NODE_MANAGER`

- **Назначение**: управляет настройкой менеджера версий Node.js во время установки
- **Значения**: `yes` или `no`
- **По умолчанию**: определяется автоматически
- **Эквивалент в CLI**: `--no-node-manager` (инвертированная логика)
- **Пример**:
  ```bash
  # Пропускаем настройку менеджера версий Node.js в CI
  curl -fsSL https://vite.plus | VP_NODE_MANAGER=no bash
  ```

### `VP_PR_VERSION`

- **Назначение**: устанавливает предварительную сборку из пулреквеста или по SHA коммита
- **Значения**: номер PR или SHA коммита
- **По умолчанию**: отсутствует
- **Подробнее**: [Предварительная сборка глобальной `vp`](/guide/upgrade#global-vp-preview)

### Переменные для разработки {#development-variables}

При разработке самого Vite+ переменные `VP_LOCAL_TGZ` (путь к локальному файлу `vite-plus.tgz`) и `VP_LOCAL_BINARY` (путь к локальному исполняемому файлу `vp`) позволяют установщику использовать локальную сборку. Установщики также самостоятельно устанавливают переменную `VP_INSTALL_STOP`; не задавайте её вручную.

## Переменные времени выполнения {#runtime-variables}

Эти переменные настраивают установленную CLI Vite+. Переменная `VP_HOME` (описанная выше) также используется во время выполнения.

### `VP_NODE_DIST_MIRROR`

- **Назначение**: URL зеркала дистрибутивов Node.js
- **По умолчанию**: `https://nodejs.org/dist`
- **Подробнее**: [Пользовательское зеркало Node.js](/guide/env#custom-node-js-mirror)

### `VP_NODE_VERSION`

- **Назначение**: переопределяет версию Node.js
- **По умолчанию**: отсутствует (определяется автоматически)
- **Пример**:
  ```bash
  # Выполняем команду с заданной версией Node.js
  VP_NODE_VERSION=22 vp env exec node -v
  ```

### `VP_NODE_SKIP_SIGNATURE_VERIFY`

- **Назначение**: пропускает проверку PGP-подписи загружаемых файлов Node.js
- **Значения**: любое непустое значение
- **По умолчанию**: отсутствует (проверка включена)
- **Подробнее**: [Проверка подписи Node.js](/guide/env#node-js-signature-verification)

### `VP_SHELL`

- **Назначение**: указывает текущую оболочку
- **По умолчанию**: определяется автоматически
- **Пример**:
  ```bash
  VP_SHELL=bash vp env print
  ```

### `VP_BYPASS`

- **Назначение**: обходит shim Vite+ и использует системный инструмент
- **Значения**: список каталогов в формате `PATH`, которые нужно обойти
- **По умолчанию**: отсутствует
- **Пример**:
  ```bash
  VP_BYPASS=/usr/local/bin node -v
  ```

### Внутренние переменные {#internal-variables}

Vite+ устанавливает дополнительные переменные `VP_*` во время обработки вызовов через shim и интеграции с оболочкой (защита от рекурсии, записи активных версий, флаги обёрток); не устанавливайте их вручную.

## Настройка TLS/CA {#tls-ca-configuration}

### `SSL_CERT_FILE` / `NODE_EXTRA_CA_CERTS`

- **Назначение**: путь к PEM-набору дополнительных сертификатов центра сертификации (`NODE_EXTRA_CA_CERTS` — соглашение Node.js)
- **По умолчанию**: системное хранилище доверенных сертификатов
- **Пример**:
  ```bash
  export SSL_CERT_FILE=/path/to/custom-ca.pem
  ```

### `VP_INSECURE_TLS`

- **Назначение**: отключает проверку HTTPS-сертификатов
- **Значения**: любое непустое значение (`1`, `true`, `yes`)
- **По умолчанию**: отсутствует (проверка включена)
- **Предупреждение**: использовать только как диагностический обходной путь; не применяйте в продакшене
- **Пример**:
  ```bash
  VP_INSECURE_TLS=1 vp env install 22
  ```

## Логирование и отладка {#logging-and-debugging}

### `VITE_LOG`

- **Назначение**: строка фильтрации логов для `tracing_subscriber`
- **По умолчанию**: отсутствует
- **Пример**:
  ```bash
  VITE_LOG=debug vp dev
  VITE_LOG=vite_task=trace vp build
  ```

### `VP_DEBUG_SHIM`

- **Назначение**: включает отладочный вывод для обработки вызовов через shim
- **Значения**: любое непустое значение
- **По умолчанию**: отсутствует
- **Пример**:
  ```bash
  VP_DEBUG_SHIM=1 node -v
  ```

## Стандартные переменные окружения {#standard-environment-variables}

Vite+ также учитывает следующие стандартные переменные окружения:

### `CI`

- **Назначение**: указывает на выполнение в окружении CI
- **Эффект**: включает тихий режим (`--yes`) для установщиков

### `NO_COLOR`

- **Назначение**: отключает цветной вывод
- **Эффект**: отключает управляющие ANSI-коды цветов

### `HOME` / `USERPROFILE`

- **Назначение**: домашний каталог пользователя
- **Эффект**: используется как основа для пути по умолчанию `~/.vite-plus`

## Приоритет {#precedence}

1. Аргументы CLI (наивысший приоритет)
2. Переменные окружения
3. Значения по умолчанию (низший приоритет)

Например, `VP_VERSION=1.0.0 vp-setup.exe --version 2.0.0` установит версию 2.0.0.
