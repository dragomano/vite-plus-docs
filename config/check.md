# Конфигурация Check {#check-config}

`vp check` выполняет форматирование, линтинг и проверку типов вместе. Блок `check` в `vite.config.ts` задаёт значения по умолчанию для составной команды, соответствующие флагам CLI `--no-fmt` и `--no-lint`.

Это полезно, когда проект хочет сохранить большую часть цепочки инструментов, но по умолчанию пропускать один из шагов. Например, команда, которая использует линтинг, но не использует форматирование, может отключить `check.fmt`, чтобы обычный `vp check` (команда, которую чаще всего запускают агенты и участники проекта) выполнял только линтинг, без необходимости помнить про `--no-fmt`.

## Пример {#example}

```ts [vite.config.ts]
import { defineConfig } from 'vite-plus';

export default defineConfig({
  check: {
    // Пропускать этап форматирования в `vp check`. По умолчанию: true.
    fmt: false,
    // Пропускать правила линтинга в `vp check`. Проверка типов по-прежнему выполняется,
    // если включены `lint.options.typeAware` и `lint.options.typeCheck`.
    // По умолчанию: true.
    lint: true,
  },
});
```

Если шаг отключён таким образом, `vp check` выводит короткую строку `note:`, чтобы было понятно, почему этот шаг не был выполнен. При конфигурации `check.fmt: false` выше:

```bash
$ vp check
note: Format skipped (check.fmt: false in vite.config.ts)
pass: Found no warnings or lint errors in 1 file (12ms, 8 threads)
```

## Область применения и приоритет {#scope-and-precendence}

- Эти параметры влияют только на составную команду `vp check`. Отдельные команды [`vp fmt`](/config/fmt) и [`vp lint`](/config/lint) не затрагиваются, поэтому при необходимости их всё равно можно запускать напрямую. Обратите внимание: любой запуск `vp check` учитывает эти настройки, включая выполнение из хука pre-commit. Если ваши задачи [`staged`](/config/staged) вызывают `vp check`, этот шаг там тоже будет пропущен.
- Шаг пропускается, если он отключён в конфигурации **или** передан соответствующий CLI-флаг. Повторно включить шаг, отключённый в конфигурации, через флаг нельзя; вместо этого нужно запускать `vp fmt` или `vp lint` напрямую.
